# index-ssl-journal

一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态网页文件的存储与展示空间。

## 项目简介

index-ssl-journal 旨在以简洁、可维护的方式托管若干独立的 HTML 文件。  
每个文件可视为一个独立的页面，内容涵盖记录、展示或信息发布等用途。  
仓库本身不涉及后端服务或动态生成逻辑，所有页面均为静态 HTML 文件。

## 目录说明

```
.
├── index.html          # 仓库根页面，列出所有归档页面链接
├── pages/              # 存放独立 HTML 页面文件的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── assets/             # 公共资源文件（如样式、脚本、图片等）
    ├── style.css
    └── script.js
```

- `index.html`：仓库入口页面，通常包含页面索引或导航信息。
- `pages/`：所有独立 HTML 页面存放于此目录，文件名按顺序或主题命名。
- `assets/`：存放页面共用的资源文件，便于维护和统一管理。

## 页面归档说明

- 每个页面均为独立的 HTML 文件，可包含完整的结构、样式和脚本。
- 页面之间无依赖关系，可单独访问或通过 `index.html` 导航。
- 归档内容遵循通用网页规范，不含外部链接或特定平台依赖。
- 页面文件命名建议使用有意义的英文或拼音，避免特殊字符。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并在 `index.html` 中添加对应链接。
- 更新页面：直接编辑对应 HTML 文件，注意保持与其他页面的兼容性。
- 删除页面：移除 `pages/` 中的文件，并同步更新 `index.html` 的链接列表。
- 资源管理：如需添加公共资源，放入 `assets/` 目录并在页面中引用。
